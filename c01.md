# Đề bài: Open Vibe Coding Challenge

**Không bị giới hạn** bởi ngôn ngữ lập trình, framework hay lĩnh vực IT cụ thể. Mục tiêu là sử dụng các công cụ AI (Cursor, Copilot, Gemini, Claude, ChatGPT...) để giải quyết một vấn đề thực tế một cách nhanh chóng và hiệu quả nhất.

---

## 1. Nhiệm vụ của bạn

Hãy tự chọn **MỘT** vấn đề mà bạn từng gặp phải trong công việc, học tập hoặc cuộc sống hằng ngày, và dùng AI để xây dựng một giải pháp cho nó.

**Ví dụ gợi ý:**

**System / Network:**
* Một script tự động dọn dẹp các Docker container/image cũ dọn rác hệ thống và gửi báo cáo qua webhook.
* Một tool CLI ping kiểm tra trạng thái sống/chết của danh sách các server định kỳ và cảnh báo nếu down.
* Một file Docker Compose tự động dựng môi trường web server (Nginx + PHP/Python) và database hoàn chỉnh.
* Một đoạn code tự động parse file log hệ thống (vd: Nginx/Apache log) và đếm số lượng lỗi HTTP `4xx/5xx`.
* Một script bash gom các file cấu hình hệ thống quan trọng trên Linux và nén thành file backup hàng ngày.

**Security:**
* Một tool Python quét nhanh các port đang mở của một IP mục tiêu và in ra các dịch vụ đang chạy.
* Một script tự động trích xuất các địa chỉ IP từ file log và kiểm tra xem có nằm trong blacklist (IP độc hại) không.
* Một tool nhỏ giả lập việc chặn truy cập mạng cục bộ vào các domain không mong muốn (ví dụ: chặn các web AI không chính thống).
* Một đoạn mã tự động sinh các rule phát hiện xâm nhập cơ bản (ví dụ cho Suricata/Snort) từ một file CSV chứa danh sách IP.
* Một script tự động kiểm tra xem độ mạnh của các mật khẩu trong một file text nhập vào có đạt chuẩn hay không.

## 2. Yêu cầu sản phẩm bàn giao

Vui lòng nộp lại các hạng mục sau:

1. **Sản phẩm hoạt động được dạng Prototype:** Source code đẩy lên GitHub/GitLab kèm theo file `README.md`. File README phải hướng dẫn chi tiết cách chạy (run/deploy) dự án này để người chấm có thể test được ngay.
2. **Tài liệu Kiến trúc:** Trình bày ngắn gọn (tối đa 1 trang):
    * Vấn đề bạn chọn giải quyết là gì?
    * Tại sao bạn chọn tech stack / công cụ này?
    * Luồng hoạt động chính (có thể dùng một sơ đồ text đơn giản hoặc mô tả flow).
3. **Nhật ký "Vibe Coding" (Bắt buộc):** Xuất (export) lịch sử chat với AI hoặc chụp lại các prompt quan trọng nhất mà bạn đã dùng. Đặc biệt chú trọng vào các prompt bạn dùng để thiết kế cấu trúc ban đầu và prompt dùng để debug khi AI viết code sai.
4. **Video Demo (Khuyến khích):** Một video quay màn hình 2-3 phút giải thích sản phẩm và chia sẻ ngắn gọn cảm nhận về quá trình "pair-programming" với AI trong dự án này.

---

## 3. Quy tắc

* **Tự do AI:** Khuyến khích sử dụng AI 100% để viết code, generate dữ liệu test, hoặc viết tài liệu.
* **Giới hạn thời gian:** Bạn có **[Điền số giờ/ngày, ví dụ: 48 giờ]** kể từ khi nhận đề bài. Khuyến cáo thời gian thực làm (effort) chỉ nên rơi vào khoảng 4-6 tiếng.
* **Tính làm chủ:** AI viết code, nhưng **BẠN** là người chịu trách nhiệm về code đó.

---

## 4. Tiêu chí đánh giá

Đánh giá dựa trên cách bạn làm việc:

* **Product Thinking (25%):** Vấn đề bạn chọn có thực tế không? Giải pháp bạn đưa ra (dù do AI code) có thực sự giải quyết trọn vẹn vấn đề đó không?
* **AI Orchestration & Prompting (30%):** Qua lịch sử chat, bạn có giao tiếp với AI rõ ràng không? Bạn có cung cấp đủ "ngữ cảnh" (context) cho AI không? Bạn chia nhỏ bài toán tốt đến mức nào?
* **Engineering & Architecture Quality (25%):** Cấu trúc dự án có gọn gàng không? Giải pháp có dễ dàng triển khai (deploy) không? Có quan tâm đến bảo mật và tối ưu hiệu năng cơ bản không?
* **Debugging & Troubleshooting (20%):** Cách bạn xử lý khi AI "ảo giác" (hallucinate) hoặc sinh ra code lỗi. Khả năng đọc hiểu log và định hướng lại AI.
